# <a name="call-resource-type"></a><span data-ttu-id="32aea-103">call 资源类型</span><span class="sxs-lookup"><span data-stu-id="32aea-103">call resource type</span></span>

[!INCLUDE [beta-disclaimer](../../includes/beta-disclaimer.md)]

<span data-ttu-id="32aea-104">当应用程序有传入呼叫或应用程序通过 `app/calls` 上的 `POST` 创建新的传出呼叫时，将创建 **call** 资源。</span><span class="sxs-lookup"><span data-stu-id="32aea-104">The **call** resource is created when there is an incoming call for the application or the application creates a new outgoing call via a `POST` on `app/calls`.</span></span>

<span data-ttu-id="32aea-105">呼叫可以设置为对等呼叫或多方呼叫。</span><span class="sxs-lookup"><span data-stu-id="32aea-105">Calls can be set up as a peer-to-peer or as a multiparty call.</span></span> <span data-ttu-id="32aea-106">若要创建或加入多方呼叫，需要提供 `chatInfo` 和 `meetingInfo`。</span><span class="sxs-lookup"><span data-stu-id="32aea-106">For creating or joining a multiparty call, supply the `chatInfo` and `meetingInfo`.</span></span> <span data-ttu-id="32aea-107">如果未提供，则会自动创建新的临时会议。</span><span class="sxs-lookup"><span data-stu-id="32aea-107">If these are not supplied, a new ad hoc meeting is created automatically.</span></span> <span data-ttu-id="32aea-108">对于传入呼叫，将这些值记录在高可用性存储中，以便在应用程序崩溃时它可以重新加入呼叫。</span><span class="sxs-lookup"><span data-stu-id="32aea-108">For an incoming call, record these values in a highly available store, so that your application to rejoin the call in the event your application crashes.</span></span>

<span data-ttu-id="32aea-109">尽管不能多次邀请相同的身份，但应用程序可以多次加入同一会议。</span><span class="sxs-lookup"><span data-stu-id="32aea-109">Although the same identity cannot be invited multiple times, it is possible for an application to join the same meeting multiple times.</span></span> <span data-ttu-id="32aea-110">每次应用程序加入时，都会为该会议呼叫提供一个不同的呼叫 `id`。</span><span class="sxs-lookup"><span data-stu-id="32aea-110">Each time the application joins, a distinct call `id` is provided for that call to the meeting.</span></span> <span data-ttu-id="32aea-111">我们建议你使用单独的身份加入会议，以便客户端将其显示为不同的参与者。</span><span class="sxs-lookup"><span data-stu-id="32aea-111">We recommend that you use separate identities to join the meeting in order for the clients to display them as different participants.</span></span>
