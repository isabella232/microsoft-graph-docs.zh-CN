# <a name="authorization-and-the-microsoft-graph-security-api"></a><span data-ttu-id="af6dd-103">授权和 Microsoft Graph 安全性 API</span><span class="sxs-lookup"><span data-stu-id="af6dd-103">Authorization and the Microsoft Graph Security API</span></span>

<span data-ttu-id="af6dd-104">可通过 Microsoft Graph 安全性 API 访问的安全数据是很敏感的，它受到权限和 Azure Active Directory (Azure AD) 角色保护。</span><span class="sxs-lookup"><span data-stu-id="af6dd-104">Security data accessible via the Microsoft Graph Security API is sensitive and protected by both permissions and Azure Active Directory (Azure AD) roles.</span></span>

<span data-ttu-id="af6dd-105">Microsoft Graph 安全性 API 支持以下两种类型的授权：</span><span class="sxs-lookup"><span data-stu-id="af6dd-105">The Microsoft Graph Security API supports two types of authorization:</span></span>

- <span data-ttu-id="af6dd-106">**应用级授权** - 没有已登录用户（例如，SIEM 方案）。</span><span class="sxs-lookup"><span data-stu-id="af6dd-106">**Application-level authorization** - There is no signed-in user (for example, a SIEM scenario).</span></span> <span data-ttu-id="af6dd-107">授权由授予给应用程序的权限决定。</span><span class="sxs-lookup"><span data-stu-id="af6dd-107">The permissions granted to the application determine authorization.</span></span> 
    ><span data-ttu-id="af6dd-108">**注意：** 此选项还可以支持基于角色的访问控制 (RBAC) 由应用程序管理的情况。</span><span class="sxs-lookup"><span data-stu-id="af6dd-108">**Note:** This option can also support cases where Role-Based Access Control (RBAC) is managed by the application.</span></span>
- <span data-ttu-id="af6dd-109">**用户委派授权** - 身份为 Azure AD 租户成员的用户已登录。</span><span class="sxs-lookup"><span data-stu-id="af6dd-109">**User delegated authorization** - A user who is a member of the Azure AD tenant is signed in.</span></span> <span data-ttu-id="af6dd-110">除了已被授予所需权限的应用程序之外，用户必须是 Azure AD 有限管理员角色的成员 - 安全读者或安全管理员。</span><span class="sxs-lookup"><span data-stu-id="af6dd-110">The user must be a member of an Azure AD Limited Admin role - either Security Reader or Securty Administrator - in addition to the application having been granted the required permissions.</span></span>

<span data-ttu-id="af6dd-111">若要从 Graph 浏览器调用 Microsoft Graph 安全性 API：</span><span class="sxs-lookup"><span data-stu-id="af6dd-111">If you're calling the Microsoft Graph Security API from Graph Explorer:</span></span>

- <span data-ttu-id="af6dd-112">Azure AD 租户管理员必须对 Graph 浏览器应用显式授予所请求的权限。</span><span class="sxs-lookup"><span data-stu-id="af6dd-112">The Azure AD tenant admin must explicitly grant consent for the requested permissions to the Graph Explorer application.</span></span>
- <span data-ttu-id="af6dd-113">用户必须是 Azure AD 中安全读者有限管理员角色的成员（安全读者或安全管理员）。</span><span class="sxs-lookup"><span data-stu-id="af6dd-113">The user must be a member of the Security Reader Limited Admin role in Azure AD (either Security Reader or Security Administrator).</span></span>

><span data-ttu-id="af6dd-114">**注意**：Graph 浏览器不支持应用级授权。</span><span class="sxs-lookup"><span data-stu-id="af6dd-114">**Note**: Graph Explorer does not support application-level authorization.</span></span>

<span data-ttu-id="af6dd-115">若要从自定义应用或你自己的应用调用 Microsoft Graph 安全性 API：</span><span class="sxs-lookup"><span data-stu-id="af6dd-115">If you're calling the Microsoft Graph Security API from a custom or your own application:</span></span>

- <span data-ttu-id="af6dd-116">Azure AD 租户管理员必须对你的应用显式授权。</span><span class="sxs-lookup"><span data-stu-id="af6dd-116">The Azure AD tenant admin must explicitly grant consent to your application.</span></span> <span data-ttu-id="af6dd-117">这对于应用程序级别授权和用户委派授权都是必需的。</span><span class="sxs-lookup"><span data-stu-id="af6dd-117">This is required both for application-level authorization and user delegated authorization.</span></span>
- <span data-ttu-id="af6dd-118">如果正在使用用户委派授权，用户必须是 Azure AD 中安全读者或安全管理员有限管理员角色的成员。</span><span class="sxs-lookup"><span data-stu-id="af6dd-118">If you're using user delegated authorization, the user must be a member of the Security Reader or Security Administrator Limited Admin role in Azure AD.</span></span>

<span data-ttu-id="af6dd-136">例如，假定你拥有一个应用程序、两个 Azure AD 租户（**T1** 和 **T2**），以及两个权限（**P1** 和 **P2**）。</span><span class="sxs-lookup"><span data-stu-id="af6dd-136">For example, assume that you have an application, two Azure AD tenants, **T1** and **T2**, and two permissions, **P1** and **P2**.</span></span> <span data-ttu-id="af6dd-137">以下是授权过程：</span><span class="sxs-lookup"><span data-stu-id="af6dd-137">The following is the authorization process:</span></span>

- <span data-ttu-id="af6dd-142">当租户 **T2** 中的用户获取应用程序的 Azure AD 令牌时，该令牌不包含任何权限，因为租户 **T2** 的管理员尚未向该应用程序授予权限。</span><span class="sxs-lookup"><span data-stu-id="af6dd-142">When users in tenant **T2** get an Azure AD token for the application, the token does not contain any permissions - because the admin of tenant **T2** did not yet grant permissions to the application.</span></span> <span data-ttu-id="af6dd-143">权限必须按照*每个租户*和*每个应用程序*进行授予。</span><span class="sxs-lookup"><span data-stu-id="af6dd-143">Permission must be granted *per tenant* and *per application*.</span></span>

- <span data-ttu-id="af6dd-150">如需使租户 **T1** 中的应用程序再次运行，租户 **T1** 的管理员必须向应用程序显式授予权限 **P1** 和 **P2**。</span><span class="sxs-lookup"><span data-stu-id="af6dd-150">To make the application work again in tenant **T1**, the admin of tenant **T1** must explicitly grant permissions **P1** and **P2** to the application.</span></span>
